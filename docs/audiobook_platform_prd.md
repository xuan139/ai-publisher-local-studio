# Audiobook Browser Platform PRD

## 1. 文档目标

本文档定义一个面向操作员的浏览器平台，用于将长文本内容生产为 audiobook，并支持生成、编辑、校对、版本管理和导出。

平台目标不是“一键出整本书”，而是提供一条稳定、可追踪、可回滚的生产链路：

- 导入文本
- 拆章拆段
- 配置发音与声音
- 批量生成 TTS
- 浏览器内试听和局部重生
- 自动质检与人工校对
- 装配章节与整本书导出

## 2. 产品定位

### 2.1 目标用户

- 内容运营
- 配音制作操作员
- 质检人员
- 项目管理员

### 2.2 核心场景

- 将小说、专栏、故事文本生产为 audiobook
- 对已生成音频进行段级修订
- 在不覆盖旧结果的前提下做版本比对
- 快速找到错读、漏读、停顿不自然的问题段

### 2.3 非目标

- 第一阶段不做完整 DAW 级音频编辑器
- 第一阶段不做自动配乐和复杂混音
- 第一阶段不做全自动多角色戏剧化演绎
- 第一阶段不做出版平台分发

## 3. 产品原则

### 3.1 Segment 是最小单元

平台的最小生产、审核和回滚单位是 `Segment`，不是整章、也不是整本书。

### 3.2 非破坏性编辑

所有修改都生成新版本，不能直接覆盖历史音频。

### 3.3 文本与音频双向追踪

每个 Segment 都必须能追踪：

- 原文
- 朗读稿
- 生成参数
- request id
- 音频版本
- 审核状态

### 3.4 先自动检查，再人工确认

平台必须先做自动质检，再交给人工校对，以降低纯人工听审成本。

## 4. MVP 范围

### 4.1 必须有

- 导入 `txt/md/docx`
- 自动拆章拆段
- 朗读稿编辑
- 单旁白声线配置
- 批量 TTS 任务下发
- 段级试听
- 改文后局部重生
- ASR 回听比对
- 问题标记
- 章节导出 `mp3`

### 4.2 第二阶段再做

- 多角色声线
- pronunciation dictionary 高级规则
- 章节片头片尾模板
- 批量重命名与批量章节设置
- 整书 `m4b`
- 多人协作审批流

## 5. 业务流程

### 5.1 标准流程

1. 新建项目
2. 导入文本
3. 自动拆章拆段
4. 操作员修订 `tts_text`
5. 选择 narrator voice 和默认参数
6. 批量生成选中章节
7. 自动执行 ASR 对齐与技术质检
8. 质检员逐段试听与标记问题
9. 对问题段做改文、换 voice 或重生成
10. 审核通过后装配章节
11. 导出章节音频和整书压缩包

### 5.2 问题段处理

1. 系统发现疑似错误
2. 操作员打开问题段
3. 查看原文、朗读稿、ASR 文本和差异
4. 决定采用以下一种处理方式：

- 改 `tts_text` 后重生
- 保留文字，只换 voice
- 保留文字，只调整语速
- 保留音频，只改剪切点
- 人工通过

## 7. 页面结构

## 7.1 项目列表页

目标：查看全部 audiobook 项目及其生产状态。

字段：

- 书名
- 作者
- 语言
- 当前阶段
- 章节数
- 已生成时长
- 待校对段数
- 更新时间

操作：

- 新建项目
- 导入文本
- 复制项目模板
- 归档项目

## 7.2 项目总览页

目标：提供书级别控制台。

模块：

- 项目摘要
- 章节树
- 任务摘要
- 质检摘要
- 导出摘要

关键指标：

- 总字数
- 预估总时长
- 已通过 Segment 数
- 待处理问题数
- 失败任务数

## 7.3 文本准备页

目标：从原始文本生成可朗读脚本。

布局：

- 左侧：章节导航
- 中间：原文编辑区
- 右侧：拆段结果和预处理建议

功能：

- 自动拆章
- 自动拆段
- 标点与断句优化
- 删除不该朗读内容
- 手工调整段边界
- 将 `source_text` 转为 `tts_text`

## 7.4 声音配置页

目标：定义默认 voice、speed、style 和 provider。

功能：

- 设置 narrator
- 试听 voice
- 保存为模板
- 章节覆盖
- 段落覆盖

## 7.5 生成工作台

目标：批量创建和监控 TTS 任务。

功能：

- 选择章节或选中 Segment 生成
- 查看排队状态
- 失败重试
- 取消任务
- 查看 provider request id
- 查看 credits 和费用估算

## 7.6 校对编辑页

目标：对 Segment 做试听、校对、重生成和版本比对。

布局：

- 左侧：章节和段落列表
- 中间：播放器、波形、版本切换
- 右侧：文本、差异、问题标签、生成参数

功能：

- 播放和循环播放
- 波形预览
- 原文/朗读稿/ASR 文本对比
- 裁掉头尾静音
- 改文后重生
- A/B 比较
- 标记问题
- 审核通过并跳下一段

## 7.7 章节装配页

目标：将已通过的 Segment 组装为章节音频。

功能：

- 调整段顺序
- 设置段间 gap
- 插入章节标题音频
- 章节试听
- 章节导出

## 7.8 导出页

目标：输出最终交付物。

支持输出：

- 每章 `mp3`
- 全书 `zip`
- 审核报告
- manifest

## 8. 角色与权限

### 8.1 Admin

- 管理项目
- 管理 voice 模板
- 查看成本
- 导出全部结果
- 管理用户与权限

### 8.2 Operator

- 编辑文本
- 发起生成
- 局部重生
- 章节装配

### 8.3 Reviewer

- 试听
- 标记问题
- 审核通过或退回

## 9. 自动质检要求

### 9.1 文本一致性检查

- ASR 回听文本与 `tts_text` diff
- 检测漏读、多读、数字错读

### 9.2 声学检查

- 削波
- 静音过长
- 响度异常
- 时长异常

### 9.3 发音检查

- 人名
- 地名
- 英文缩写
- 数字日期

## 10. 非功能要求

### 10.1 性能

- 列表页在 1,000 个 Segment 规模下可流畅筛选
- 单段波形预览在 2 秒内可见
- 任务状态刷新延迟不超过 5 秒

### 10.2 可靠性

- 任务失败后可重试
- 音频产物必须可追溯到文本和参数版本
- 任一段失败不阻塞整章其他段生成

### 10.3 可审计性

- 保留操作日志
- 保留每次生成参数和请求 id
- 保留审核结果和审核人

## 11. 核心指标

- 文本导入成功率
- 首次生成成功率
- 平均每千字生成耗时
- 自动质检命中率
- 单段平均返工次数
- 章节一次通过率
- 单本书平均生产成本

## 12. 里程碑建议

- 文本导入
