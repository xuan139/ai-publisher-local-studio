# AI Publisher 審核員一頁操作說明

適用對象：音訊審核、校對、質檢同事

## 你的任務

檢查每一段音訊是否可進入最終匯出：

`打開 Review -> 播放段落 -> 對照 tts_text -> 標記問題 -> 通過或退回`

## 使用前準備

- 瀏覽器打開 [http://127.0.0.1:8000](http://127.0.0.1:8000)
- 建議登入帳號：`reviewer@example.com`
- 建議登入密碼：`review123`
- 若需跨流程協助，可使用 `admin@example.com / admin123`；此 admin 帳密保持不變
- 其他預設角色帳號見 [預設角色帳號](default_accounts.md)
- 專案內已經有生成完成的音訊段落

## 審核步驟

### 1. 進入專案的 `Review` 頁面

先選擇要審的章節與段落，確認該段已有可播放音訊。

### 2. 先聽，再看文字

播放當前段落，重點檢查：

- 是否有明顯錯讀
- 是否漏字、加字
- 節奏是否自然
- 停頓是否合理
- 聲音是否穩定

### 3. 對照 `tts_text`

以 `tts_text` 為準，不要只看原文。

如果音訊和 `tts_text` 不一致，優先判斷：

- 是文字本身要改
- 還是音訊要重生

### 4. 標記問題

有問題時，建立 issue 或直接退回。

常見問題類型：

- 發音錯誤
- 漏讀 / 多讀
- 節奏不自然
- 停頓異常
- 內容與文字不一致

### 5. 做出審核結論

- 沒問題：標記為 `已通過`
- 有問題：標記為 `已退回`

只有 `已通過` 的段落才會進入後續章節匯出。

## 什麼情況應該退回

- 關鍵詞、人名、地名唸錯
- 與 `tts_text` 差異明顯
- 明顯卡頓、斷句錯誤
- 單段品質差到不適合直接上線

## 什麼情況可以直接通過

- 內容與 `tts_text` 基本一致
- 只有輕微語氣差異
- 不影響理解與整體聆聽體驗

## 常見提醒

- 如果是文本本身不適合朗讀，先回去改 `tts_text`
- 不要因為小問題就重做整章，通常只退回當前段落
- 審核目標是「可發布」，不是要求每段完全機械一致

## 一句話版

先聽音訊，再對照 `tts_text`，有問題就退回，沒問題就通過，最後讓通過段落進入匯出。
