# Audiobook Platform Web Design Overview

## 1. 设计目标

Web 设计的核心目标不是“炫”，而是让操作员能长时间高效处理大量 Segment。界面必须同时满足：

- 长时间使用不疲劳
- 复杂信息高可读
- 问题定位快
- 重生成和回滚成本低

## 2. 设计方向

采用 `production console` 风格，而不是营销站风格。

关键词：

- 冷静
- 稳定
- 高信息密度
- 明确状态
- 最小干扰

## 3. 信息架构

一级导航建议：

- Projects
- Pipeline
- Review
- Voices
- Exports
- Settings

书内二级导航建议：

- Overview
- Text Prep
- Voice Setup
- Generate
- Review
- Assemble
- Export

## 4. 布局框架

### 4.1 全局布局

- 左侧固定导航栏
- 顶部上下文栏
- 中央工作区
- 右侧可收起详情侧栏

### 4.2 页面模式

页面分三种模式：

- `管理型`：项目列表、任务列表
- `编辑型`：文本准备、voice 配置
- `审核型`：波形试听和版本对比

## 5. 关键页面布局建议

## 5.1 项目列表页

组件：

- 顶部搜索栏
- 状态筛选器
- 批量操作栏
- 高密度表格

重点强调：

- 状态标签
- 待处理数量
- 更新时间

## 5.2 项目总览页

布局：

- 顶部摘要卡片
- 左侧章节树
- 中部进度与任务图
- 底部问题段列表

摘要卡片建议：

- 总字数
- 预估总时长
- 生成进度
- 审核进度
- 失败任务
- 预计成本

## 5.3 文本准备页

布局：

- 左侧章节树
- 中间双栏对照编辑器
- 右侧规则和建议面板

中间双栏：

- 左：`source_text`
- 右：`tts_text`

这样操作员能很快判断哪些内容该朗读，哪些不该读。

## 5.4 生成工作台

布局：

- 顶部工具栏
- 中间任务表格
- 底部任务日志

重点功能区：

- 批量选择
- 一键生成
- 一键重试失败
- request id 查看

## 5.5 校对编辑页

这是平台最重要的页面，建议作为专门优化对象。

布局：

- 左栏：段列表
- 中栏：播放器和波形
- 右栏：文本、差异、版本、问题标签

交互顺序必须顺手：

1. 选段
2. 自动开始播放
3. 同时展示 `tts_text` 和 ASR diff
4. 快捷键标记问题
5. 立即重生或通过
6. 自动跳到下一段

## 5.6 章节装配页

布局：

- 左侧 Segment 顺序区
- 中部章节试听和拼装时间轴
- 右侧渲染配置

这里不做复杂 DAW，只做：

- 排序
- gap 调整
- 插入章节标题
- 整章试听

## 6. 视觉系统

## 6.1 配色建议

不要用纯黑纯白，也不要用紫色平台默认风格。

建议色板：

- 背景：暖灰白 `#F4F1EA`
- 面板：象牙白 `#FBF8F2`
- 主文本：深墨蓝 `#1F2937`
- 次文本：石墨灰 `#6B7280`
- 主操作色：暗青绿 `#0F766E`
- 警告色：铜橙 `#C2410C`
- 错误色：砖红 `#B91C1C`
- 成功色：苔绿 `#3F6212`

用途原则：

- 绿色只表示“已通过”
- 红色只表示“必须处理”
- 黄色或橙色表示“需人工确认”

## 6.2 字体建议

不要走系统默认风格堆砌。

建议：

- 英文与数字 UI：`IBM Plex Sans` 或 `Source Sans 3`
- 中文 UI：`Noto Sans SC` 或 `Source Han Sans SC`
- 表格和日志：等宽字体 `IBM Plex Mono`

## 6.3 密度策略

这个平台属于高密度工具，不应过度留白。

建议：

- 列表采用 `compact` 模式
- 表单采用 `comfortable` 模式
- 审核页采用 `focus` 模式

## 7. 组件设计

关键组件：

- 状态标签 `Status Pill`
- 波形播放器 `Waveform Player`
- 文本差异查看器 `Diff Viewer`
- 音频版本切换器 `Take Switcher`
- 章节树 `Chapter Tree`
- 段落队列 `Segment Queue`
- 批量任务表格 `Job Table`

### 7.1 状态标签

统一颜色和文案：

- `draft`
- `ready`
- `queued`
- `generating`
- `review required`
- `approved`
- `rejected`
- `exported`

### 7.2 Diff Viewer

建议规则：

- 删除内容用红色删除线
- 新增内容用绿色底色
- 可疑错读用橙色描边

### 7.3 Waveform Player

最少提供：

- 播放
- 暂停
- 循环
- 拖动定位
- 头尾 trim 标记
- 倍速播放

## 8. 交互原则

### 8.1 快捷键优先

校对页一定要支持快捷键：

- `Space` 播放暂停
- `A` 通过
- `R` 退回
- `G` 重生
- `N` 下一段
- `P` 上一段

### 8.2 默认动作尽量减少点击

- 进入段时自动载入最新 take
- 可选自动播放
- 审核通过后自动跳下一段

### 8.3 先显示问题，再显示细节

在列表里先让人看到：

- 哪一段有问题
- 问题类型
- 严重程度

不要先把所有文本铺满。

## 9. Web 响应式策略

主要使用场景仍然是桌面端。

断点建议：

- `>= 1440px`：完整三栏
- `1024px - 1439px`：双栏，右侧可抽屉
- `< 1024px`：仅保留列表和简单试听，不作为主生产界面

结论：

- 平台应优先为桌面浏览器设计
- 移动端只做查看和轻审核

## 10. 页面状态设计

每页都要有这四种状态：

- 初始空状态
- 加载状态
- 无结果状态
- 错误状态

示例：

- 无 Segment 时引导用户先拆段
- 无 voice 时引导先配置 narrator
- 无生成结果时给出一键生成入口

## 11. 可视化建议

平台里只保留有价值的图表：

- 章节进度条
- 每日生成量
- 返工率
- 问题类型分布
- provider 成功率

不要放无意义的大图或装饰性图表。

## 12. 首版前端技术建议

- `Next.js`
- `TypeScript`
- `Tailwind CSS` 或 `CSS Modules`
- `TanStack Table`
- `TanStack Query`
- `wavesurfer.js`
- `Zustand` 或 `Redux Toolkit`

## 13. 设计验收标准

设计方案应达到以下标准：

- 操作员可以在 3 次点击内打开任一问题段
- 审核员可以在单页内完成试听、标注和重生
- 批量任务和段级任务状态一眼可分辨
- 旧版本音频可在 10 秒内找到并对比
- 视觉层级清晰，不依赖说明文才能操作
