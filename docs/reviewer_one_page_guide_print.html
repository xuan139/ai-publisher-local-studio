審核員一頁操作說明

適用於音訊審核與校對同事。你的工作是判斷每一段是否可直接進入最終匯出。

審核口訣 先聽音訊 -> 對照 tts_text -> 標記問題 -> 通過或退回

審核步驟

  1. 進入 Review：打開專案後，進入 `Review` 頁面，選擇章節與段落。
  2. 確認有音訊：當前段落必須已有可播放 take，否則先回生成流程。
  3. 先聽再看：播放完整段落，先感受是否有錯讀、卡頓、異常停頓或聲音不穩。
  4. 對照 tts_text：以 `tts_text` 為準，檢查是否漏字、多字、讀錯關鍵詞。
  5. 判斷問題來源：如果文字本身不適合朗讀，應先回去改 `tts_text`；如果是聲音輸出問題，退回重生。
  6. 記錄問題：對有問題的段落建立 issue，寫明類型與簡短描述。
  7. 做出結論：可接受就標記 `已通過`；不可接受就標記 `已退回`。

登入帳號

建議使用 `reviewer@example.com / review123`。若需跨流程協助，可使用 `admin@example.com / admin123`；此 admin 帳密保持不變。

常見問題類型

發音錯誤 漏讀 / 多讀 節奏不自然 停頓異常 內容不一致

審核畫面

`Review` 頁面是審核核心。以段落播放器、文字對照與問題標記為主，不需要重做整章。

判定原則

  • 關鍵詞、人名、地名唸錯，直接退回。
  • 只有輕微語氣差異，但不影響理解，可直接通過。
  • 有問題時優先退回單段，不要擴大成整章重做。
