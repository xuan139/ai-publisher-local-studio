# 漫画生产流程设计草案

## 目标

在现有 `AI Publisher Local Studio` 中新增一条 `comic / motion comic` 工作流，复用现有的：

- 专案管理
- 文本导入
- 角色资料
- 生成任务
- 审核
- 导出

## 当前实现状态

截至当前版本，漫画 Phase 1 已完成第一批落地：

- `projects.project_type` 已支持 `comic / motion_comic`
- 已新增并接通：
  - `comic_scripts`
  - `comic_pages`
  - `comic_panels`
- 前端已上线：
  - `漫畫腳本`
  - `分鏡工作台`
  - `畫格生成`
  - `頁面排版`
- 侧边栏已将 `有聲書流程` 与 `漫畫流程` 分开

当前仍未完成的部分：

- 场景 / 道具资产库
- 页面级正式导出
- 配音 / 配乐 / 音效时间轴
- 动态漫画与视频生成

并在此基础上补齐漫画制作所需的：

- 剧本
- 漫画脚本
- 分镜
- 角色视觉资产
- 场景与道具资产
- 画格生成
- 字幕 / 气泡 / 拟声字
- 配音 / 配乐 / 音效
- 页面装配与导出

## 建议流程

### Phase 1：静态漫画主流程

1. 项目立项
   - 选择项目类型：`audiobook / comic / motion_comic / video`
   - 设定语言、题材、目标读者、页幅比例、风格模板

2. 故事大纲 / 剧本
   - 输入原始故事、章节、人物关系、世界观
   - 生成或编辑章节级故事大纲
   - 确定每章的冲突、高潮、结尾钩子

3. 漫画脚本
   - 把章节剧本转换成 `页 -> 格 -> 台词 -> 旁白 -> 动作说明`
   - 每格保存：
     - 场景
     - 出场角色
     - 情绪
     - 对白
     - 拟声字
     - 构图意图

4. 分镜头
   - 为每页每格生成镜头语言：
     - 景别
     - 机位
     - 镜头方向
     - 构图
     - 光线
     - 节奏
   - 输出为可编辑的 storyboard 卡片

5. 角色与资产
   - 角色资料沿用现有 `character_profiles`
   - 角色形象图沿用现有 `character_looks`
   - 新增场景与道具资产库，保存统一视觉参考

6. 画格生成
   - 按分镜和角色参考图生成 panel 图像
   - 保留多版本、可比较、可回退
   - 支持局部重绘和重试

7. 页面排版
   - 将 panel 组合成页面
   - 支持对白气泡、旁白框、拟声字、页码、章节标题
   - 输出页面预览

8. 审核与返修
   - 审核项：
     - 剧情连续性
     - 角色一致性
     - 构图可读性
     - 字幕溢出
     - 错字
     - 分镜节奏
   - 沿用现有 `review_issues`

9. 导出
   - `PNG / JPG / PDF / ZIP`
   - 章节导出
   - 全书导出

### Phase 2：动态漫画 / 有声漫画

10. 配音
    - 复用现有 `voice_profiles` 与 `segments`
    - 台词、旁白、角色声线一一对应

11. 配乐与音效
    - 为页、格或镜头段落配置：
      - BGM cue
      - 环境音
      - 拟音
      - 转场音

12. 时间轴装配
    - 页面停留时长
    - 镜头切换节奏
    - 音频与字幕同步
    - 导出 `MP4 / MOV / HLS`

## 建议补充的模块

除了你提到的 `剧本 / 脚本 / 分镜头 / 角色 / 配乐 / 配音`，我建议一起加上这 4 个模块：

1. 世界观与风格圣经
   - 统一题材、服装、时代、色彩、光线、镜头语言

2. 场景与道具资产库
   - 否则角色一致了，背景和物件会一直漂

3. 页面排版 / 气泡系统
   - 这是漫画成品感最关键的一层，不能只停在单张图生成

4. 返修版本管理
   - 每次重写脚本、重做分镜、重生画格都需要版本链

## 最适合这个仓库的落地顺序

建议不要一次把全部做满，而是按下面顺序推进：

1. 先做 `静态漫画`
   - 剧本
   - 脚本
   - 分镜
   - 角色 / 场景资产
   - 画格生成
   - 页面排版
   - PDF / ZIP 导出

2. 再做 `有声漫画`
   - 角色配音
   - 旁白
   - 音效
   - 配乐
   - 时间轴

3. 最后做 `动态漫画 / 视频版`
   - 镜头动画
   - 运镜
   - 转场
   - 短视频导出

## 推荐结论

可以做，而且很适合在这个项目里做。

最好的方式不是把它当成“给漫画多加几个设置项”，而是把它做成一条完整的 `Comic Production Workflow`：

`故事 -> 漫画脚本 -> 分镜 -> 角色/场景资产 -> 画格生成 -> 页面排版 -> 审核 -> 导出`

如果要带声音，再在这个主流程之上叠加：

`配音 -> 配乐/音效 -> 时间轴 -> 动态导出`
